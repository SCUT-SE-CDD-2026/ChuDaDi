# 字体规范

## 1. 目标

本项目当前阶段的字体目标是：

- 建立统一、稳定、可落地的字体系
- 服务于“现代、流畅、轻拟物、克制”的整体视觉风格
- 优先保证游戏界面的识别效率、信息层级和完成度
- 避免字体混乱、气质割裂、局部随意选择字重

本阶段以游戏界面为核心进行统一，测试性质的主页面仅保证接入同一套规范，不做额外风格投入。

## 2. 字体资源

当前资源目录为 `app/src/main/res/font`。

### 2.1 英文无衬线

- `inter_18pt_extrabold.ttf`
- `inter_18pt_medium.ttf`
- `inter_24pt_extrabold.ttf`
- `inter_24pt_medium.ttf`

### 2.2 英文衬线

- `lora_bold.ttf`
- `lora_medium.ttf`

### 2.3 中文黑体

- `source_han_sans_cn_heavy.otf`
- `source_han_sans_cn_medium.otf`

### 2.4 中文宋体

- `source_han_serif_cn_heavy.otf`
- `source_han_serif_cn_medium.otf`

## 3. 本阶段结论

本阶段统一采用“无衬线主导”的字体策略。

- 普通 UI 文本使用无衬线体系
- 标题与强调文本使用无衬线粗字重
- 扑克牌牌值与花色也统一使用无衬线体系
- 当前阶段不启用衬线字体作为正式界面字体

这样做的原因是：

- 更符合当前项目追求的现代、干净、稳定气质
- 更容易与轻拟物、结构化界面统一
- 中英数字、状态信息、操作按钮更容易形成一致的视觉系统
- 牌值与花色能与整体 UI 语言保持一致，避免局部风格跳脱

## 4. 字体角色映射

### 4.1 主 UI 字体

主 UI 字体用于绝大多数界面文字。

- 中文：`SourceHanSansCN`
- 英文、数字、拉丁字符：`Inter`

适用范围：

- 页面标题
- 模块标题
- 按钮文字
- 玩家名称
- 状态提示
- 说明文本
- 弹层文本
- 结果页文本
- 卡牌牌值与花色

### 4.2 保留字体

以下字体暂时保留在资源中，但本阶段默认不使用：

- `Lora`
- `SourceHanSerifCN`

只有在未来出现明确的展示型页面、品牌页、特殊插画型标题需求时，才重新评估是否启用。

## 5. 字重映射规则

当前阶段只使用两档主字重，避免层级失控。

### 5.1 Medium

用于：

- 正文
- 辅助信息
- 状态文本
- 次要标签
- 普通按钮文案
- 对局过程中的说明性文字

对应资源：

- 中文：`source_han_sans_cn_medium.otf`
- 英文：`inter_18pt_medium.ttf` 或 `inter_24pt_medium.ttf`

### 5.2 Heavy / ExtraBold

用于：

- 页面标题
- 面板标题
- 主操作按钮文字
- 扑克牌牌值
- 需要强化识别的关键数字或关键信息

对应资源：

- 中文：`source_han_sans_cn_heavy.otf`
- 英文：`inter_18pt_extrabold.ttf` 或 `inter_24pt_extrabold.ttf`

### 5.3 使用约束

- 不随意引入第三档、第四档字重
- 不通过过多字重变化制造层级
- 层级优先靠字号、颜色、间距、容器层级表达
- 同一类组件内部的字重必须一致

## 6. 文本层级映射

以下规则用于映射到 Compose `Typography` 以及后续自定义文本样式。

### 6.1 一级标题

用于：

- 页面主标题
- 结果页主标题
- 重要弹层标题

规则：

- 使用无衬线粗字重
- 强调清晰和稳定，不追求装饰性

建议映射：

- `headlineMedium`
- `headlineSmall`

### 6.2 二级标题

用于：

- 区块标题
- 面板标题
- 重点区域名

规则：

- 使用无衬线粗字重
- 比正文更强，但弱于一级标题

建议映射：

- `titleLarge`
- `titleMedium`

### 6.3 正文

用于：

- 页面说明
- 状态文本
- 普通描述
- 面板内容

规则：

- 使用无衬线中等字重
- 保持较高可读性和稳定行高

建议映射：

- `bodyLarge`
- `bodyMedium`
- `bodySmall`

### 6.4 标签与按钮

用于：

- 小标签
- 操作按钮文字
- 席位信息辅助字段
- 次级状态信息

规则：

- 小标签默认中等字重
- 主按钮可使用粗字重
- 按钮文案优先强调清晰，不使用过紧字距

建议映射：

- `labelLarge`
- `labelMedium`
- `labelSmall`

## 7. 游戏界面专项规则

### 7.1 扑克牌牌值与花色

本项目第一版正式规定：牌值和花色统一使用无衬线字体。

规则如下：

- 牌值使用无衬线粗字重
- 花色使用无衬线粗字重或中偏粗视觉效果
- 红黑花色依靠颜色区分，不依靠字体风格区分
- 牌值与花色以“识别优先”为第一原则
- 不为牌面单独引入书法、装饰、拟古、花体字体

设计理由：

- 保持现代、干净、克制的整体风格
- 与当前 UI 风格一致
- 降低牌面局部“复古感”对整体风格的干扰
- 提升小尺寸下的识别稳定性

### 7.2 玩家信息区

用于：

- 玩家名
- 剩余牌数
- 当前回合状态

规则：

- 玩家名可略高于辅助信息层级
- 剩余牌数与状态文本使用较小一级文本
- 同一个席位卡片内不超过两档字号层级

### 7.3 操作区按钮

规则：

- 主按钮文案使用粗字重无衬线
- 次按钮与弱化按钮默认中等字重无衬线
- 所有按钮文案保持简短清楚，避免通过字重制造攻击性视觉感受

### 7.4 提示与反馈文本

规则：

- 行动提示、错误提示、状态提示统一使用无衬线正文体系
- 不通过夸张字体处理突出提示
- 提示层级主要依靠容器、颜色、位置和阴影表达

## 8. 中英混排规则

项目中存在中文、英文、数字、扑克牌字符混排场景，因此必须遵守以下规则：

- 中文文本优先使用 `SourceHanSansCN`
- 英文和数字优先使用 `Inter`
- 同一段文本中保持统一的无衬线风格
- 不在同一组件内同时混用无衬线和衬线体系
- 牌值、数字、状态文本尽量保持稳定宽度感，避免视觉跳动

## 9. 组件映射表

当前阶段建议使用以下映射作为实现基准。

| 场景 | 字体体系 | 字重 | 备注 |
| --- | --- | --- | --- |
| 页面主标题 | 无衬线 | Heavy / ExtraBold | 如主页标题、结果页标题 |
| 区块标题 | 无衬线 | Heavy / ExtraBold | 如面板标题、区块名 |
| 正文说明 | 无衬线 | Medium | 默认正文 |
| 状态提示 | 无衬线 | Medium | 如等待中、已过牌 |
| 主按钮文字 | 无衬线 | Heavy / ExtraBold | 保证识别力度 |
| 次按钮文字 | 无衬线 | Medium | 与主按钮拉开层级 |
| 玩家名称 | 无衬线 | Medium 或 Heavy | 视视觉层级决定 |
| 剩余牌数 | 无衬线 | Medium | 辅助信息 |
| 牌值 | 无衬线 | Heavy / ExtraBold | 核心识别元素 |
| 花色 | 无衬线 | Heavy / ExtraBold | 通过颜色区分红黑 |
| 结果信息 | 无衬线 | Medium / Heavy | 标题粗，内容中等 |

## 10. 实现要求

代码实现时，应将以上规范落到统一主题系统中，而不是在页面里零散写死。

要求如下：

- 优先通过 `MaterialTheme.typography` 统一常规文本样式
- 对扑克牌牌值与花色，可补充单独的牌面文本样式
- 页面内避免直接临时指定字体，除非是牌面等明确的特殊场景
- 同类文本尽量复用同一主题样式，避免样式漂移

## 11. 当前阶段边界

本规范当前只解决“字体选择与映射规则”问题，不单独解决以下问题：

- 全量布局自适配策略
- 全量颜色体系重构
- 所有页面的完整视觉升级

但后续所有 UI 改造都必须以本规范为前提，不再重新讨论基础字体方向。
