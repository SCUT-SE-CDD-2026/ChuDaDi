# Highlight And Shadow Design

适用于需要表达“浮起感”“选中态”“聚焦态”的 UI 元素，尤其是卡片、卡牌、浮层等具有明确轮廓的对象。

## 核心原则

1. 高光和阴影必须贴合对象轮廓，不要使用与主体形状不符的发光形状。
2. 高光和阴影优先服务于体积感与层级感，不是装饰边框。
3. 高光应从主体附近向外逐渐变透明，阴影应从主体附近向外逐渐变淡。
4. 选中态应同时提升高光、阴影和层级，而不是只加一圈描边。
5. 未选中态也应保留轻微基础阴影，避免元素显得过于平面。

## 形状规则

1. 主体和高光、阴影一般情况下都是相同形状的。例如主体是圆角矩形时，高光和阴影也应以圆角矩形为基础。否则容易出现形状不贴合的问题。
3. 如果需要柔和发光，使用“单层模糊后的主体轮廓”，而不是多层硬叠加的伪渐变。

## 强度关系

1. 选中高光强于未选中高光；未选中态通常不应有明显高光。
2. 选中阴影强于未选中阴影。
3. 高光的模糊范围通常应大于阴影的模糊范围。
4. 阴影颜色应低饱和、低亮度，避免脏黑；高光颜色应较亮但不过分发白。

## 数值选用经验

1. 优先使用相对主体的比例，而不是先写绝对值。范围、偏移、包络尺寸都应先按主体宽高推导。
2. 高光包络通常可从主体尺寸的 `1.08x - 1.18x` 起步，阴影包络通常略小于高光或与其接近。
3. 对竖向主体，纵向扩展通常应明显大于横向扩展，否则容易出现左右过宽、上下不足的问题。
4. 高光透明度可先从 `0.35 - 0.65` 起步；阴影透明度通常可从 `0.18 - 0.35` 起步。若阴影颜色本身较深，可使用更低透明度。
5. 高光 blur 通常应大于阴影 blur，先确定范围和形状，再微调透明度，不要反过来。
6. 如果局部数值调整无效，先怀疑上层约束链，而不是继续放大 blur 或 alpha。

## 比例经验

1. 外层光影包络应相对主体略放大，不要等比无脑放大。
2. 纵向余量通常应大于横向余量，尤其是竖向主体。
3. 高光范围应略大于阴影范围或至少不小于阴影范围。
4. 阴影的方向偏移应单独控制，不要只靠不对称拉伸去模拟方向。

## 布局约束经验

1. 高光和阴影是否可见，先看父容器约束，再看局部参数。
2. 如果调整局部宽高无效，优先检查上层区域的高度上限、裁切和分区分配。
3. 高光与阴影需要独立的显示安全区，不能默认与主体共用同一块空间。
4. 先确认真正生效的上层约束，再判断局部参数是否为主控开关。

## 实现建议

1. 不要把“固定透明度背景 + 描边”当成高光主体。
2. 高光更适合使用单层、连续、模糊化的轮廓发光。
3. 阴影更适合使用单层、方向明确、强度克制的模糊阴影。
4. 多层叠加不能作为近似实现。
5. 当需要真实柔光时，自定义绘制模糊形状，而不是继续堆叠多层效果。

## 排查顺序

1. 先确认高光/阴影层是否真的拥有尺寸。
2. 再确认父容器是否裁切或限高。
3. 再确认高光/阴影是否被相邻元素覆盖。
4. 最后才调整颜色、透明度、模糊半径和范围。

## 常见误区

1. 把描边误当成高光。
2. 用多层硬叠加伪装渐变，结果像包了一层壳。
3. 只调局部 glow 尺寸，不检查父级约束是否允许显示。
4. 未选中态完全没有基础阴影，导致元素太平。
5. 选中态只增加亮度，不提升层级和阴影。
