# 仓库初始化指南

## 1. 适用对象

本文档面向第一次拉取本仓库的团队成员，说明本地需要准备哪些环境、初始化顺序是什么，以及和前端 `bun install` 类项目的差异。

## 2. 拉取仓库后还需要做什么

需要。仓库里已经提交了项目级配置，但每个成员本地仍需准备自己的开发环境。

仓库已提供：

- Gradle Wrapper
- Android Gradle 配置
- 版本目录 `gradle/libs.versions.toml`
- detekt / ktlint / lint 配置
- 项目源码和脚本

每个成员仍需本地准备：

- Android Studio
- 可用 JDK
- Android SDK
- Git
- 本机代理方案（如果访问 Google Maven 较慢）
- 本地 Git hook 安装

## 3. 为什么不需要手动安装依赖

- 本项目没有单独的“安装依赖”命令
- Gradle 会在首次执行同步、构建或检查任务时自动解析并下载依赖
- 这和 TS / Bun 项目通常需要先执行 `bun install` 下载依赖不同

也就是说，本项目的“安装依赖”通常隐含在这些命令里：

```bash
./gradlew.bat help --no-daemon --console=plain
./gradlew.bat tasks --no-daemon --console=plain
./gradlew.bat assembleDebug --no-daemon --console=plain
```

## 4. 推荐初始化步骤

### 4.1 安装基础环境

- 安装 Android Studio 稳定版
- 安装 Android SDK
- 使用 Android Studio 自带 JDK，或本机可用 JDK
- 安装 Git

### 4.2 打开项目

- 用 Android Studio 打开仓库根目录
- 等待 Gradle Sync 完成
- 若 IDE 仍有脚本红线，手动执行一次 `Sync Project with Gradle Files`

### 4.3 确认 Android SDK

- 首次打开后，Android Studio 一般会生成本机 `local.properties`
- 该文件只记录本机 SDK 路径，不应提交到版本库
- 若缺少对应平台版本，请在 SDK Manager 中安装项目所需平台

### 4.4 运行基础验证命令

```bash
./gradlew.bat --version
./gradlew.bat help --no-daemon --console=plain
./gradlew.bat tasks --no-daemon --console=plain
./gradlew.bat lint
./gradlew.bat detekt
./gradlew.bat ktlintCheck
./gradlew.bat test
```

### 4.5 构建调试包

```bash
./gradlew.bat assembleDebug --no-daemon --console=plain
```

## 5. 网络与代理说明

- 命令默认不携带代理参数，方便每位成员按自己的环境使用
- 如果本地访问 Google Maven 较慢或超时，需要自行补充代理环境变量
- 不要把个人代理端口和代理方案写死进团队默认命令

## 6. 安装本地 pre-commit hook

仓库中的 `scripts/pre-commit.sh` 会被版本管理，但 `.git/hooks/` 不会随 Git 自动分发。

因此，新成员拉取仓库后，需要在本机安装 hook。推荐直接运行仓库里的安装脚本：

### Windows

```bash
scripts\install-hooks.bat
```

### Git Bash / macOS / Linux

```bash
sh scripts/install-hooks.sh
```

安装完成后，本地 `git commit` 会自动执行：

- `ktlintFormat`
- `detekt`

## 7. 常见问题

### 7.1 为什么我拉完仓库就能直接跑 `./gradlew.bat`？

因为依赖由 Gradle 在任务执行时自动下载，不需要单独运行 `bun install` 这一类命令。

### 7.2 为什么 hook 没有自动生效？

因为 `.git/hooks/` 是本地 Git 目录，不受版本库管理。每个成员都需要在自己的机器上安装一次。

### 7.3 如果 pre-commit 失败怎么办？

先看终端输出，再按需要执行：

- `./gradlew.bat ktlintFormat`
- `./gradlew.bat detekt`
- `./gradlew.bat lintFix`

修复后重新提交即可。
