# 页面清单

本文档基于 `软件综合开发实训启动纲要` 与 `Android 蓝牙锄大地游戏技术架构文档` 整理，覆盖当前目标版本需要支持的三类核心玩法：

- 蓝牙联机对局
- AI 人机对局
- 规则配置与结果结算闭环

页面设计原则：

- 主界面统一承载所有玩法入口
- 房主负责联机房间配置与开局控制
- 对局页同时支持房主、蓝牙玩家、AI 玩家三类身份展示
- 设置页只影响后续新开局，不强行打断当前对局
- 联网游戏（云服务器）不属于当前阶段核心目标，仅保留入口占位

---

## 1. 页面总览

### 当前阶段需要完整设计并实现的页面

1. 主界面
2. 蓝牙房间页
3. 蓝牙搜索/加入页
4. 对局页
5. 结果页
6. 设置页
7. 规则说明页

### 当前阶段只预留占位的页面

1. 联网游戏大厅页（占位）
2. 联网房间页（占位）
3. 排行榜/战绩页（可选占位）

## 2. 页面详细设计

### 主界面

- UI：
  - 左上角玩家信息区：玩家头像、玩家昵称
  - 游戏标题 / Logo
  - 创建房间按钮
  - 加入房间按钮
  - 联网游戏按钮（开发中）
  - 设置按钮
  - 说明按钮（规则、玩法介绍）
  - 可选信息区：最近一次使用的规则、音效状态、版本号
- 功能：
  - 点击“创建房间”进入蓝牙房间页，并以房主身份初始化房间
  - 点击“加入房间”进入蓝牙搜索/加入页
  - 点击“联网游戏”进入联网游戏占位页
  - 点击“设置”进入设置页
  - 点击“说明”进入规则说明页
- 说明：
  - 主界面是所有玩法的统一入口
  - AI 人机配置不单独设入口页，而是由房主在房间页通过空位添加 AI 的方式完成
  - 入口命名要直接对应用户任务，避免先进入一个抽象的“模式选择页”再二次跳转

### 蓝牙房间页

- UI：
  - 标题栏：房间标题、返回按钮、房主标识
  - 房间信息区：房主设备名、当前规则、当前人数 `x/4`
  - 四人槽位区：
    - 固定显示 4 个位置
    - 已占用位置显示玩家或 AI 的头像、昵称、类型、连接状态/难度信息
    - 未占用位置显示虚线边框和 `+` 标志
    - 房主点击空位后弹出 AI 难度选择界面
    - 选择难度后在对应空位加入该难度 AI
    - 房主可以踢人
  - 规则操作区（仅房主可见）：
    - `当前:南方规则/北方规则` 切换按钮
    - 开始游戏按钮
    - 可选：移除已添加 AI 的入口
  - 成员等待区（仅成员可见）：
    - `已连接: [房主设备名]`
    - `等待房主开始游戏`
  - 通用提示区：
    - 蓝牙可见性状态
    - 连接中/已连接/连接断开提示
    - 人数不足提示
- 功能：
  - 房主进入该页时自动请求蓝牙广播/可见性相关权限，并进入可被发现状态
  - 房主创建房间后开始监听客户端连接
  - 房主可点击空位补入 AI，并为该 AI 选择难度
  - 房主可在房间页切换南方规则/北方规则，并向后续客户端同步当前规则
  - 房主仅在 4 个槽位都被真人或 AI 占满，且真人成员全部准备后才能点击开始游戏；条件不足时给出提示
  - 成员进入该页后不具备规则切换权和开局权，只负责查看当前连接状态并等待开始
  - 成员需要显式准备；若有人未准备或退出，则不能直接开始下一局
  - 房主点击开始游戏后，所有已连接成员同步跳转到对局页
  - 对局结束后统一返回该页，可保留当前房间成员、AI 配置与累计比分，由房主重新组织下一局
  - 若有成员断开连接，房主侧成员列表实时更新，并重新校验是否满足开局条件
- 说明：
  - 蓝牙联机采用明显的“房主-成员”非对称流程，房主和成员虽然可以复用同一页面组件，但界面状态和可操作能力必须区分
  - AI 人机配置并入房间页处理，避免额外新增独立配置页
  - 房间页需要同时兼容两种来源：蓝牙加入的真人成员，以及房主手动补入的 AI

### 蓝牙搜索/加入页

- UI：
  - 标题栏：返回按钮、页面标题
  - 蓝牙状态区：蓝牙是否开启、权限状态、刷新按钮
  - 设备列表区：
    - 附近可加入设备列表
    - 每个设备显示 `[设备名],[MAC]`
  - 操作区：
    - 开启蓝牙按钮（必要时）
    - 搜索设备按钮
  - 对话框：
    - 点击某个设备后弹出“发起蓝牙连接”确认框
    - 提供“连接”按钮
  - 提示区：搜索中、连接中、连接失败原因、已连接状态
- 功能：
  - 进入页面后自动开始扫描，并请求蓝牙扫描相关权限
  - 检查本机蓝牙开关状态
  - 搜索附近可加入的房主设备
  - 点击设备后弹出连接确认框，确认后发起连接
  - 连接成功后进入蓝牙房间页（成员视角），显示“已连接房主”和“等待房主开始游戏”
  - 连接失败时给出明确提示并允许重试
- 说明：
  - 该页职责是“搜索并连接”，不是完整房间页
  - 成员在该页完成的是入房动作，真正的开局控制仍然由房主在蓝牙房间页完成

### 对局页

- UI：
  - 顶部信息区：
    - 当前阶段
    - 当前出牌者
    - 当前轮牌型 / 上一手牌说明
    - 当前局累计分数或排名摘要
    - 房间模式标识（蓝牙 / AI）
  - 中部牌桌区：4 个玩家座位
    - 自己：底部展示完整手牌，可点击选中/取消选中
    - 其他玩家/AI：顶部/左右展示头像、昵称、剩余手牌数、最近动作（过 / 出牌 / 等待）
    - 当前行动玩家高亮
  - 中央出牌区：
    - 当前桌面牌组
    - 上一手有效出牌说明
    - 回合结束或清桌提示
  - 底部操作区：
    - 提示按钮（可选）
    - 过牌按钮
    - 出牌按钮
    - 重选按钮
  - 浮层/弹窗：
    - 非法出牌提示
    - 轮到谁出牌提示
    - 回合倒计时（如 15s）
    - 断开连接提示（蓝牙模式）
    - 对局结束弹窗（或直接跳转结果页）
- 功能：
  - 初始化牌局：洗牌、发牌、确定首个出牌玩家
  - 玩家选牌/取消选牌
  - 校验牌型合法性与是否大过上家
  - 执行出牌、过牌、轮转到下家
  - AI 自动决策并出牌
  - 蓝牙模式下接收远端玩家指令并同步桌面状态
  - 一轮结束后清空桌面比较基准，进入新一轮
  - 判断有人出完牌后结束本局
  - 将结果传递到结果页
- 必须覆盖的规则与交互能力：
  - 基础牌型识别：单张、对子、三张、五张组合（顺子、同花、葫芦、铁支、同花顺）
  - 基础比牌逻辑
  - 南北规则切换支持
  - 不同玩家身份统一展示：本地真人、蓝牙真人、AI 玩家
- 说明：
  - 对局页是所有玩法的统一承载页，不应为“蓝牙对局”和“AI 对局”拆成两套完全独立界面
  - 差异主要体现在控制来源：本地点击、蓝牙消息、AI 决策
  - 若房间中只有房主和 AI，则该页实际表现为人机对局；若房间中同时有蓝牙成员和 AI，则表现为混合房间对局

### 结果页

- UI：
  - 本局胜负结果 / 名次 / 分数变化
  - 玩家结果列表：
    - 玩家名
    - 玩家类型（真人 / AI）
    - 剩余手牌数
    - 本局得分
    - 累计总分
  - 操作区：
    - 返回房间按钮（蓝牙模式）
- 功能：
  - 展示本局最终结果与累计比分
  - 蓝牙模式下点击“返回房间”回到蓝牙房间页，由房主重新组织下一局
  - 不提供独立“再来一局”按钮；“再来一局”的实际流程就是返回房间后重新准备并由房主再次开始。退出对局需要在房间页退出，并且清除本房间的积累战绩数据。
  - 保留同一房间内的累计胜负记录和积分
- 说明：
  - 结果页既承担单局结算，也承担多人联机场景下的阶段过渡
  - 蓝牙模式下，房主和成员在结果后的回流逻辑也不同：房主负责重新开局，成员负责等待

### 设置页

- UI：
  - 规则设置区：
    - 南方规则 / 北方规则切换
    - 可扩展的特殊规则开关（可选）
  - 音画设置区：
    - 音效开关
    - 背景音乐开关
    - 日间 / 夜间主题切换
    - 动画开关（可选）
  - 对局偏好区：
    - 默认出牌倒计时
- 功能：
  - 自动保存到本地（DataStore / SharedPreferences）
  - 影响后续新开局，不实时打断当前对局
  - 房主创建房间时读取当前设置作为默认房间规则
  - 默认出牌倒计时应限制为大于 5 秒
- 说明：
  - 设置页负责“长期偏好”，不负责承载某一局临时房间细节
  - 房间专属配置应由房主在蓝牙房间页确认和广播

### 规则说明页

- UI：
  - 玩法简介
  - 基础流程
  - 牌型说明
  - 规则差异（南北规则）
  - 联机与 AI 模式说明（可简要）
- 功能：
  - 纯展示
  - 支持滚动阅读
- 说明：
  - 该页既服务普通玩家，也可作为课程展示时的玩法说明页面

### 联网游戏大厅页（占位）

- UI：
  - 标题
  - “云服务器联机功能开发中”提示
  - 返回按钮
- 功能：
  - 暂不实现真实联网逻辑
- 说明：
  - 为第四阶段的云联机能力预留入口，避免未来主界面结构大改

### 联网房间页（占位）

- UI：
  - 房间信息占位
  - 玩家列表占位
  - 准备/开始按钮占位
- 功能：
  - 暂不实现

### 排行榜/战绩页（可选占位）

- UI：
  - 历史战绩占位
  - 胜率/总分占位
- 功能：
  - 暂不实现，可在后期接入本地统计或联网账户体系

## 3. 页面跳转关系

### 主流程

- 主界面 -> 创建房间 -> 蓝牙房间页
- 主界面 -> 加入房间 -> 蓝牙搜索/加入页 -> 蓝牙房间页
- 主界面 -> 设置页
- 主界面 -> 规则说明页
- 主界面 -> 联网游戏大厅页（占位）

### 蓝牙联机流程

- 蓝牙房间页 -> 对局页 -> 结果页 -> 蓝牙房间页
- 蓝牙房间页 -> 退出 -> 主界面
- 蓝牙搜索/加入页 -> 连接失败 -> 留在当前页重试

### 蓝牙联机角色流程

- 房主：主界面 -> 创建房间 -> 蓝牙房间页（监听连接、切换规则、开始游戏） -> 对局页 -> 结果页 -> 蓝牙房间页
- 成员：主界面 -> 加入房间 -> 蓝牙搜索/加入页（搜索、连接） -> 蓝牙房间页（等待房主开始） -> 对局页 -> 结果页 -> 蓝牙房间页

### AI 对局流程

- 主界面 -> 创建房间 -> 蓝牙房间页（房主给空位添加 AI） -> 对局页 -> 结果页 -> 蓝牙房间页

## 4. 设计补充约束

- 对局页路由建议统一，只通过参数区分 `gameMode = bluetooth / ai`
- 当前参考流程下，蓝牙房间默认按 4 名真人玩家设计，是否引入 AI 补位应视后续需求单独扩展
- 房主拥有房间规则确认权与开局权，成员没有规则切换权和开局权
- 房主拥有 AI 补位权：可点击空位并选择 AI 难度，把 AI 加入指定槽位
- 蓝牙模式下的所有关键状态变更应由房主裁定并广播，客户端只负责展示与发送操作请求
- 房间开始条件应为：4 个位置全部占满，且真人成员全部准备完成
- AI 在系统中应被视为一种特殊玩家控制器，页面展示上与真人玩家尽量统一
- 当前版本不建议把“联网游戏”混入现有联机逻辑实现，避免影响蓝牙联机与本地 AI 主线
